# Digital Zen API - Краткая инструкция

Простое руководство по развертыванию API для сохранения данных пользователей.

## 🚀 Быстрый старт (5 шагов)

### 1. Генерация секретного ключа

Перейди на https://randomkeygen.com/ и скопируй один из "Fort Knox Passwords".

Пример: `k8jH#mP9$nQ2*rT5&vW7@xY0!zA3^bC6`

### 2. Настройка бэкенда

1. В папке `api/` скопируй `config.example.php` → `config.php`
2. Открой `config.php`
3. Заполни:
   ```php
   define('DB_PASS', 'твой_пароль_от_базы');
   define('API_SECRET_KEY', 'k8jH#mP9$nQ2*rT5&vW7@xY0!zA3^bC6');
   ```

### 3. Загрузка файлов на хостинг

Загрузи эти файлы на `digital-zen.csmpoint.com`:

```
digital-zen.csmpoint.com/api/
├── .htaccess
├── config.php
├── helpers.php
└── user
```

**ВАЖНО: Установи правильные права доступа!**

После загрузки файлов через FTP или File Manager:

- Папка `api/`: права `755`
- Файлы `.php`: права `644`

Без этого получишь ошибку 403 Forbidden!

### 4. Создание таблиц в базе

1. Зайди в **phpMyAdmin**
2. Выбери базу `u387418961_digital_zen_db`
3. Вкладка **SQL**
4. Скопируй весь текст из файла `api/database.sql`
5. Вставь и нажми **Go**

### 5. Настройка расширения

Открой `src/modules/common/constants/api-config.const.ts`:

```typescript
export const API_CONFIG: ApiConfig = {
  apiUrl: 'https://digital-zen.csmpoint.com/api',
  apiKey: 'k8jH#mP9$nQ2*rT5&vW7@xY0!zA3^bC6', // ТОТ ЖЕ ключ что и в config.php
};
```

## 🆘 Проблемы?

### "Invalid API key" в расширении

**Причина**: Ключи не совпадают

**Решение**:

1. Проверь `api/config.php` - скопируй ключ
2. Проверь `api-config.const.ts` - вставь ТОЧНО такой же ключ
3. Пересобери: `npm run build`

### 403 Forbidden (HTML ошибка)

**Это самая частая проблема!** Сервер блокирует доступ к PHP файлам.

**Быстрое решение:**

1. **Проверь права доступа** (самая частая причина):
   - Папка: `755`
   - PHP файлы: `644`

2. **Загрузи файл `.htaccess`** в папку api

3. **Используй полный путь к файлу**:
   - ✅ `https://digital-zen.csmpoint.com/api/user`
   - ❌ `https://digital-zen.csmpoint.com/api/`

**Подробное руководство:** [TROUBLESHOOTING_403.md](./TROUBLESHOOTING_403.md)

**Другие причины:**

- ModSecurity блокирует (свяжись с техподдержкой хостинга)
- Старая версия PHP (нужна 7.4+)
- Сервер не разрешает .htaccess (свяжись с техподдержкой)

### "Failed to connect to database"

**Причина**: Неверный пароль или настройки базы

**Решение**:

1. Проверь пароль в `config.php`
2. Убедись что база `u387418961_digital_zen_db` существует
3. Проверь что таблицы созданы через phpMyAdmin

### Таблицы не создались

**Причина**: SQL не выполнился

**Решение**:

1. Убедись что выбрана правильная база данных
2. Посмотри на ошибки в phpMyAdmin
3. Попробуй выполнить SQL построчно

## 📖 Структура базы данных

### Таблицы:

- **users** - пользователи (email, user_id)
- **periods** - периоды фокусировки
- **websites** - заблокированные сайты
- **focused_times** - временные интервалы

## 🎯 Что умеет API

### GET /api/user

Получить данные пользователя

**Параметры**: `user_email` или `user_id`

### POST /api/user

Сохранить данные пользователя

**Тело запроса**:

```json
{
  "user_email": "user@example.com",
  "user_id": "123456",
  "periods": [...]
}
```

## 📝 Формат данных

API автоматически конвертирует данные между Angular и MySQL:

- **Периоды** → таблица `periods`
- **Сайты** → таблица `websites`
- **Время фокуса** → таблица `focused_times`

Вся логика конвертации в `api/user` - код простой и понятный!
